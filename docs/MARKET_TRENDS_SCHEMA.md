# MF Pulse — Market Trends Schema

Daily, structured, historically-retained market intelligence. Generated by
`scripts/build_snapshots.py` (snapshots) and `scripts/daily_intelligence.py` (what-changed),
written to append-only `data/warehouse/*.jsonl` and mirrored to the `fact_daily_*` tables.

## fact_daily_market_snapshot (1 row/day)
`snapshot_date, total_schemes, active_schemes, trend_ready_count, risk_ready_count,
research_ready_count, top_category_30d, top_amc_30d, top_fund_30d, market_breadth,
avg_return_30d, avg_volatility_90d, created_at`

## fact_daily_category_snapshot (1 row/day/category)
`snapshot_date, category, fund_count, avg_return_30d, avg_return_90d, avg_health_score,
positive_fund_pct, top_fund_code, worst_fund_code, created_at`

## fact_daily_amc_snapshot (1 row/day/AMC)
`snapshot_date, amc, active_fund_count, avg_return_30d, avg_health_score, top_fund_code,
worst_fund_code, metadata_coverage_pct, created_at`

## fact_daily_intelligence (append-only insight log)
`intelligence_date, intelligence_type, entity_type, entity_id, title, summary, severity,
source_metric, previous_value, current_value, created_at`

### Intelligence types (deterministic diffs of consecutive snapshots)
`new_top_fund`, `enter/exit_top_decile`, `category_leader_change`, `amc_leader_change`,
`risk_warning` (breadth drop / volatility rise), `breadth_improved`, `metadata_improved`,
`stale_warning`, `factsheet_acquired`. Every insight cites a `source_metric` with
`previous_value` → `current_value`. **No LLM; nothing is fabricated.**

## Generation
- Snapshots: idempotent per day (re-run replaces that day, prior days retained).
- Intelligence: requires ≥2 snapshots; diffs the latest two and appends only real changes.
- Both run in `scripts/factsheet_pipeline.py` and the monthly cron; snapshots should also run
  in the daily NAV cron (one snapshot per trading day) to build the historical series.
